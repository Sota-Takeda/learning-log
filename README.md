# Learning Log（学習ログアプリ）

学習内容を **学習日** と **学習時間（分）** で記録し、
検索・集計しながら学習の振り返りができる学習ログアプリです。

未経験エンジニアのポートフォリオとして、
**認証・CRUD・バリデーション・認可・検索・集計・UI設計** まで一通り実装しました。

---

## アプリ概要
日々の学習を記録し、
「いつ・どれくらい学習したか」を振り返れることを目的としたアプリです。

単なる CRUD 実装で終わらせず、
**実務を意識した設計・安全性・ユーザー体験**を重視して開発しました。

---

## 主な機能
- ユーザー登録 / ログイン / ログアウト（Laravel Breeze）
- 学習ログの CRUD
- 作成 / 一覧 / 詳細 / 編集 / 削除
- バリデーション（FormRequest）
- 認可（Policy）
- 他人の学習ログにはアクセス不可（403）
- 学習ログ検索（タイトル / メモ）
- ページネーション
- 学習時間の集計
- 今日の学習時間合計
- 共通レイアウトによる UI 統一
- 論理削除（Soft Delete）
- 削除済み一覧 / 復元

---

## 工夫した点・設計意図

### 認証
Laravel Breeze を利用して認証機能を実装しました。
認証処理を自作せず、**セキュリティと開発効率を重視**しています。

---

### バリデーション（FormRequest）
バリデーションは Controller に書かず、
FormRequest に切り出すことで責務を分離しました。

Controller では処理の流れのみが分かる構成にし、
可読性と保守性を意識しています。

---

### 認可（Policy）
Policy を用いて、
**他人の学習ログにはアクセスできない**よう制御しています。

URL を直接叩かれても不正操作できないようにし、
403 を返すことでデータの安全性を担保しています。

---

### 検索・ページネーション
一覧画面では、タイトル・メモを対象とした検索機能を実装しました。
検索条件を保持したままページ遷移できるようにし、
ユーザー体験を意識しています。

---

### リファクタリング（責務の整理）
検索・絞り込み・並び順といったクエリ処理は
Model の scope に切り出しました。

Controller では処理の流れのみを記述し、
データ取得の詳細は Model に集約することで、
可読性と再利用性を高めています。

---

### 学習時間の集計
当日の学習ログから
**今日の学習時間の合計**を算出して一覧画面に表示しています。

CRUD だけでなく、
データを活用した表示を意識しました。

---

### UI / UX の工夫
共通レイアウトを用いて全画面でナビゲーションを統一しました。
保存・更新・削除時にはフラッシュメッセージを表示し、
操作結果が分かりやすい UI を意識しています。

---

### テスト
Feature テストを追加し、以下を確認しています。

- 未ログイン時のアクセス制御
- 認可（他人のデータ操作は 403）
- 入力バリデーションの動作確認

---

### Soft Delete（論理削除）
実務を想定し、学習ログは物理削除ではなく
論理削除（deleted_at）を採用しました。

削除済み一覧から復元できるようにし、
誤削除への耐性を持たせています。

---

## 開発を通して意識したこと
- まず機能実装を優先し、その後 UI や導線を整理する
- Controller を肥大化させない設計
- 実務で使われる構成・考え方を意識する
